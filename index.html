Android & CMS



작품 요약

주변 지인의 제안으로 안드로이드 플랫폼과 관리자 페이지를 만드는 외주를 받아 프로젝트를 진행한 경험이었습니다. 앱을 간단히 소개하자면 사용자가 블랙박스 혹은 핸드폰 동영상 촬영을 하고 이 영상을 가지고 어플리케이션을 통해 신고를 합니다. 이에 따라 관리자는 관리자 웹 페이지에서 사용자들이 신고한 내역의 상태를 변경하거나 신고된 정보를 기반으로 그래프 or 신고 지역에 대한 정보를 시각화해서 보여주는 프로그램입니다.

어플리케이션의 주요 기능

- 동영상 편집
- CMS에서 사용자 신고내역 변경에 따른 FCM 알림
- 신고 내역 갔고오는 경우의 txt 파일 데이터 파싱모듈 개발
- Android Laoyout, RecyclerView, Viewpager, FrameLayout, Google Map, Glide 이미지 처리 등
- 통신은 Retrofit Library를 사용
- 각 종 Validation (ex 차량번호, 이메일 형식, 핸드폰 번호 형식, EditText 입력 유무 등)

관리자페이지 주요 기능

- 자동 로그인 관리
- 구글지도 연동 및 사용자 신고정보를 갖고 와서 해당 좌표에 마커 표시
- 신고정보를 기반으로한 그래프 처리
- Excel Export 기능과 신고영상 다운로드
- 사용자 신고정보를 기반으로한 테이블 구성 및 사용자 신고내역 상태 변경
- AJAX 통신 및 각 종 validation (예를 들어 현재 시간 기준 -10시간으로 실시간 신고접수 불러오는 경우나 인터넷 연결이 끊겼을 경우의 상태변경 등)

개발 후기

  안드로이드 작업에서는 동영상 편집을 하는데 있어서 대부분의 시간을 사용하였습니다. 그리고 핸드폰 기종마다 해상도도 달라지고 지원하는 Level도 달라져서 다양한 기종에서 테스트를 해야하고 앱의 시나리오에 있어서 다양한 루트가 존재하여 예상치 못한 에러가 나올 수 있다는점을 배웠습니다. 즉, 단순히 앱이 동작만 하는것이 아닌 다양한 case의 validation을 걸어야하고 마감처리의 중요성을 배웠습니다. 또한 처음으로 FCM 알림을 사용하였고 Retrofit 통신 등 다양한 경험을 할 수 있었던 프로젝트였습니다.
  지금까지 웹 작업은 server side rendering으로 작업을 하다가 이번에는 html/css/js/jquery로 작업을 하고 통신이 필요할 때 ajax통신을 사용하였습니다. 또한 구글지도 연동 및 그래프처리 테이블형식 등 좀 더 퀄리티 있는 개발을 하여 많은 것을 배울 수 있었습니다.

Inha Cms Page

JSP & Servlet



  학교 선배의 추천으로 인하대학교 홍보팀 관리자페이지를 만들어보자는 제의를 받았고 마침 이전 학기에 '인터넷 프로그래밍'수업에서 JSP & Servlet에 대해서 배운적이 있어 처음으로 개발자로써 돈을 받고 일을 진행한 경험이었습니다. 지금까지 한 프로젝트들은 동작만 하면되지라는 느낌으로 개발을 했다면, 이번 프로젝트의 경우에는 다양한 validation에 신경을 써야 했고 해상도마다 다르게 프로그래밍을 하였습니다. 또한 서버 환경 구축(Tomcat & Maria DB & Java 환경설정 설치 및 연동)에서 시간을 굉장히 많이 썻지만 그래도 공들인 만큼 환경설정인 면에서 얻는것도 많았던 프로젝트라고 생각합니다.

  로그인을 하지 않은 상태에서 대문 관리 페이지 혹은 회원 관리 페이지로 갈 경우의 session을 통해서 가지못하도록 validation을 걸었고, 대문 관리 페이지에서는 Drag&Drop을 통해 데이터베이스에 있는 사진의 순서를 바꾸도록 진행하였습니다. 뿐만 아니라 파일 태그를 이용하여 파일을 다운로드 및 삭제를 가능하게 하였습니다. 그리고 회원 관리 페이지에서는 유저의 정보를 테이블 형식으로 불러와 출력하게 했고, 유저정보를 추가 및 수정 및 삭제를 가능하게 하였습니다.

개발 기간 : 2019 02~03
개발 환경 : AWS Tomcat 서버 & Jsp/Servlet & MariaDB

배달의 민족 카피캣 프로젝트

Android & PHP



  방학기간 동안 평소에 관심있던 배달의 민족 어플리케이션을 디자이너와 함께 카피캣해서 만들어보자는 취지에서 시작하였습니다. 카피캣 프로젝트이다 보니 설계 과정이나 아이디어에서 시간을 쏟기 보다는 최대한 배달의 민족 어플과 비슷한 Flow에서 개발에 초점을 맞춰서 진행하였습니다. 처음으로 디자이너와 같이 협업을 진행하였으며, 클라이언트에서는 최대한 디자이너 요청사항에 맞게 개발을 하였습니다. 또한 서버도 더 이상 Local에서 진행하는 것이 아닌 AWS 서버에 Nginx를 올렸고 REST API에 최대한 맞게 진행하였습니다.

코드에 대한 자세한 설명은 github에서 작성하였습니다.

개발 기간 : 2019 01~02
클라이언트 개발 환경 : Android Platform
서버 개발 환경 : AWS Nginx PHP MySQL
