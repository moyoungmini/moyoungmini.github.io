Impolice

Android & CMS & PHP



작품 요약

주변 지인의 추천으로 안드로이드 어플리케이션과 관리자 웹 페이지를 만드는 외주를 받아 프로젝트를 진행하였습니다. 프로젝트에 대해 간단히 소개를 하자면 사용자가 가상의 경찰이 되어 사람들이 하는 불법 행위에 대한 영상을 가지고 어플리케이션을 통해 신고를 합니다. 이때 사용자가 신고한 신고 횟수가 일정량을 채울 경우 관리자를 통해 리워드를 받는 시스템입니다. 관리자는 사용자가 신고한 신고내역을 관리자 웹 페이지에서 신고한 영상을 가지고 판단하여 신고처리 여부를 결정 할 수 있습니다. 저가 이 프로젝트를 맡은 가장 큰 이유는 다음과 같습니다. 사용자가 영상을 통해 신고하는 공수가 있지만 리워드를 통해 보상을 받을 수 있고 사회적 측면에서는 리워드를 제공하지만 빈번히 일어나는 불법 행위가 줄어드는 효과가 도출됩니다. 즉, 어느 누구도 손해보는 것이 아닌 사용자와 사회적 측면 모두 긍정적 효과를 볼 수 있는 프로젝트라고 생각했기 때문입니다.

- 동영상 편집
- 웹 페이지에서 신고내역 변경에 따른 실시간 FCM 알림
- 블랙박스 영상에서 주어지는 txt 파일 데이터 파싱 개발
- Android Laoyout, RecyclerView, Viewpager, FrameLayout, Glide 등 Android UI 구성
- Google Map 3rd party API 사용
- Android - Retrofit Http 통신
- Paging 기법 사용

- Session을 통한 자동 로그인 관리
- Google map 3rd party API (javascript) 연동 및 신고정보에 대한 좌표 마커 표시
- 신고상태에 대한 그래프 처리
- Excel export 기능
- 신고정보 목록 테이블 구성 & 사용자 신고상태 실시간 변경
- WEB - Ajax 통신

Android 개발 후기

프로젝트 기획 단계에서 기술적 한계를 검토할때 동영상 편집부분 구현에 대한 고민이 있었습니다. 처음부터 동영상 편집 부분을 구현하자니 공수가 너무 많이 들고 구현 가능성에 대해서도 확실하지 않았기 때문입니다. 그래서 기존에 동영상 편집 소스가 있는지를 먼저 검토했고 그 결과 MIT license로 되있던 코드를 찾을 수 있었습니다. 하지만 원래 기획했던 디자인 및 기능과는 차이가 많이 나는 소스 코드였기 때문에 기존 소스코드에서 많은 변경이 필요한 작업이었습니다. 하지만 소스코드를 분석하면서 좀 더 나은 code pattern을 공부 할 수 있었고 이로 인해 그 전까지의 프로젝트의 경우 디자인 패턴이나 코드에 대해 관심없이 기능적인 구현에만 신경을 썻지만 이 프로젝트부터 디자인패턴(이 프로젝트는 MVC 패턴 적용)과 유지보수를 대비하여 코드 리팩토링 등에 신경을 쓰기 시작했습니다.

이 후에는 디자이너가 준 디자인대로 작업을 시작했고 UI/UX적인 코드 구현에 있어서는 큰 어려움 없이 프로젝트를 진행 할 수 있었습니다. 하지만 이 프로젝트는 다양한 기종의 기기들이 사용 할 수 있기 때문에 Q/A 기간에 다양한 기종에 대한 테스트를 하면서 예상치 못한 버그가 나왔으며 개발 할 당시에 신경 못 썻던 앱 시나리오가 나와 다시 한번 validation의 중요성을 배웠습니다. 또한 FCM 알림을 사용하였으며 외부에서 주어지는 txt 파일에서 위도 / 경도를 뽑아 와 google map에 표시하는 작업을 수행하였습니다.

server와 통신 시 로그인 한 이후에는 access-token 값을 가지고 통신을 하였으며 GET/POST/PATCH 등의 method와 함께 http 통신을 진행하였습니다. 프로젝트 진행 도중 많은 프로젝트들에서 서버와 통신하는 코드들이 비슷하고 같은 작업임에도 불구하고 공수가 매번 든다는점을 생각하였습니다. 그래서 이러한 과정을 줄이고자 MVC 패턴 & 서버 통신 & token 관리 등 프로젝트마다 자주 사용하는 코드들을 탬플릿화 하여 만들었습니다.

CMS 개발 후기

server side rendering이 아닌 html/css/js/jquery로 front-end 작업을 한 후 서버와의 통신이 필요 할때 ajax 통신을 하도록 구현하였습니다. 또한 session을 통해 로그인을 관리하였으며 관리자 로그인 없이 url을 통해 해당 페이지로 갈 경우 로그인 페이지로 다시 이동하도록 validation을 처리하였습니다.

서버로부터 x축에 대한 y축 좌표 정보를 읽어 온 후 그래프에 표시하였으며 y축 최대 높이는 읽어 온 y값들 중 최대 값 + n(그래프에 따라 차이)으로 설정하였습니다. 또한 서버로부터 신고내역을 읽어 와 테이블에 표시를 하였으며 상태관리항목에서 신고상태를 변경 할 수 있습니다. 이때 서버로 신고상태 변경정보를 알리고 서버는 해당 유저에게 FCM알림을 통해 알리는 과정을 거치게 됩니다. 신고 테이블의 row에 있는 download 항목을 누를 경우 파이어베이스 연결된 url로 연결되어 동영상을 확인 할 수 있으며 테이블 최상단에 있는 항목을 클릭 시 순서대로 재 정렬을 하게 구현하였습니다. excel-export의 경우 해당 소스가 워낙 많았기 때문에 코드 커스텀 하는 과정을 좀 더 쉽게 작업을 할 수 있었습니다.

구글에서 제공하는 지도 API를 사용하였고 서버로 부터 위/경도 신고종류에 대한 데이터를 읽어와 해당 좌표에 마커표시를 띄우는 작업을 수행하였습니다.

Server 개발 후기

AWS EC2 위에서 작업을 수행하였으며 서버는 Nginx 데이터베이스는 MySQL 개발 언어는 PHP를 사용하였습니다. 로그인 및 신고 데이터 처리 등은 기존에 해왔던 것들과 유사하여 쉽게 작업할 수 있었습니다. 사용자가 어플케이션에서 신고를 했을때 해당 동영상을 Google Firebase에 저장하고, 해당 url을 서버 데이터베이스에 저장하는 방식으로 진행하였습니다.
ajax 통신 시 cors 문제가 생겼었고 서버에서 헤더에 allow-control 설정을 통하여 해결하였습니다. 서버 개발 중 데이터베이스에 대한 접근 횟수를 줄이고자 노력하였고 (SQL을 여러번 호출하는 것이 아닌 다양한 SQL을 조합한 구문 사용) 사용자가 자신의 동영상 정보에 데이터를 마이페이지에서 불러올떄 서버에서 한 번에 모든 데이터를 불러오는 것이 아닌 10개씩 끊어서 데이터 정보를 보내주게 처리하여 네트워크 사용량을 조절하는데 신경을 썻습니다.

작품 요약

학교 선배의 추천으로 인하대학교 홍보팀 관리자 페이지를 만들어보자는 제의를 받았습니다. 이전 학기에 '인터넷 프로그래밍' 학부 수업에서 JSP & Servlet에 대해 배웠고 학교 관련된 프로젝트를 진행하면 좀 더 보람을 느낄것이라 생각되어 프로젝트를 진행하였습니다. 프로젝트를 간단히 소개하자면 안드로이드 어플리케이션에서 인하대학교에 대한 정보를 소개하고, 관리자페이지에서 관리자는 사용자를 관리하며 어플리케이션 배경화면에 보여지는 이미지 순서를 변경하는 작업을 수행합니다. 이때 저가 맡은 분야는 CMS page 구현 및 웹서버 구축이었습니다.

Project 주요 기능

- 웹 서버 (AWS 위에 Apache & MariaDB) 구축
- 이미지 Drag & Drop을 통한 순서 변경
- 테이블을 통한 사용자 관리
- session을 통한 로그인 관리

개발 후기

이번 프로젝트에서 가장 크게 어려움을 느낀것은 개발환경 설정이었습니다. 지금까지 했던 작업들은 로컬에서 동작하는 것들이었지만 이 프로젝트의 경우 AWS에서 할당 받은 웹서버로 코드를 올려야 했기 때문에 웹서버에 대해서 설정할것이 많았습니다. 개발 환경 설치 후 구현하는 단계에 있어서는 큰 어려움은 없었습니다. session을 통해 로그인 관리를 유지하였으며 Drag & Drop Event가 일어날 경우 Database에서 이미지 순서를 변경하는 작업을 수행하였습니다. 또한 file 태그를 이용하여 파일을 웹서버로 업로드 및 삭제를 수행하였습니다. 또한 유저에 대한 정보를 테이블 형식으로 보여줬으며 웹의 특징 상 다양한 해상도를 가진 기기들이 접근 할 수 있으므로 모든 width/height들은 %로 작업을 하였고 글씨 크기는 em으로 정하고 작업을 시작하였습니다.

인하대학교를 통학하는 학생들 대부분은 주안역에서 511버스를 타고 등/하교를 합니다. 그러나 급한 볼일이 있거나 막차시간 혹은 수업시간이 얼마 안 남았을 경우에는 버스를 타기에 시간이 부족합니다. 이유는 등/하교 시간에는 사람들이 버스정류장에 많이 모이며 버스를 타면 아무래도 학교까지 혹은 역까지 직행으로 가는것이 아닌 돌아서 가기 때문 입니다. 그래서 대부분의 학생들은 이러한 경우에 택시를 자주 이용합니다. 하지만 학생들 대부분은 택시를 혼자서 이용을 합니다. 여러명이서 같이 택시를 이용한다면 비용 절감이 될텐데 왜 혼자 택시를 타는지에 대한 고민한 결과 이유는 다음과 같다고 생각을 했습니다. 첫번째로는 같이 택시를 타고 싶어도 누가 택시를 타고 싶어하고 그 사람의 목적지가 어디인지를 모르는점이고 두번째로는 목적지를 알더라도 그 사람이 신뢰할 수 있는지를 모르는점입니다. 저희는 이러한 2가지 이유를 택시를 이용하고 싶어하는 사람들끼리 소통할 수 있도록 도움을 주는 커뮤니케이션 장을 제공하고 학생증 인증방식으로 신뢰성유무 또한 해결해주는 어플리케이션을 제작하였습니다.

Project 주요 기능

- Google Cloud Vision API를 이용한 텍스트 추출과 이미지 편집 기능
- 3rd party API Kakao Map, Kakao Keyword rest API 사용
- Firebase에서 제공하는 Real time database를 사용한 채팅 기능
- 안드로이드 & 웹 플랫폼 프로그램 제공
- 안드로이드(Retrofit 통신) & 웹 (Ajax 통신)
- Nginx Server, MySQL DB, PHP Language를 이용한 서버 구축

개발 후기

저가 이번 프로젝트에서 맡은 개발적인 역할은 안드로이드에서는 3rd party API 구현, Google Cloud Vision API를 이용한 텍스트 추출, 이미지 편집이었고 CMS 페이지 구현 또한 맡았습니다. 이번 해커톤에서 어려움을 느꼈던점은 git 충돌과 프로젝트 계획이었습니다. 저희 팀원들과 일을 분배할 떄 최대한 Activity 별로 구별을 했지만 각 클래스에서 사용하는 많은 공유변수가 있어 정확히 화면별로 태스크를 나누기는 힘들었습니다. 저희는 이러한 git 충돌을 sourcetree tool을 이용함으로써 좀 더 쉽게 해결하였습니다. 그리고 이번 해커톤 주제는 인하대학교에 도움이 되는 프로그램 제작이었는데 프로젝트를 계획하는데 어려움을 느꼈습니다. 지금까지 대부분의 프로젝트에서 개발위주로 포지션을 맡아 진행하여 프로젝트 개발 첫 단계인 기획단계부터 어떻게 진행할지에 대해 막막했지만 '소프트웨어 공학' 과목을 수강하면서 배운 프로젝트 모델링 하는 과정(기획->설계->개발->테스트->배포)을 생각하면서 프로젝트 타켓, 사용 이유, 이 프로그램을 대체할 만한것이 있는지 등 끊임 없이 기획단계를 생각하며 진행하여 조금 더 쉽게 프로젝트를 모델링 하였습니다.

프로젝트 소개

저는 영어공부를 하면서 영어를 읽고, 쓰기는 좀 더 수월하게 진행하였지만 외국인과 말하고 듣는것에는 항상 어려움이 있었습니다. 저는 그러한 문제점을 저가 영어공부를 할때 발음에 신경을 쓰지 않아서라고 생각을 하였습니다. 또한 저는 영어공부를 주기적으로 실행하는 방법을 생각해보다가 알람기능과 함께 사용한다면 어느정도의 강제성으로 지속적으로 공부할 수 있다고 생각을 하였습니다. 그래서 저는 이러한 방법을 사용하여 영어발음을 좀 더 향상시킬 수 있도록 도와주는 어플리케이션을 기획하기 시작하였습니다. 쉽게말하면 주기적인 알람을 통해 영어발음 공부를 도와주는 어플리케이션입니다. 알람이 울리면 정확한 영어단어 발음을 사용해야 알람이 해지되게 하였으며 만약 제한시간 이내에 풀지 못할 경우 5분후 다시 알람이 울리게끔 만들었습니다. 화면별로 소개를 하면 다음과 같습니다.
-1) 메인화면 : 메인화면은 알람들을 리스트업 해주는 화면으로 알람에 대한 ON/OFF 설정, 알람이 울리는 요일 및 시간과 알람 삭제(스와이프를 통한 삭제)를 할 수 있습니다.
-2) 환경설정화면 : 이런알람에 테마 색 및 알람에 대한 전체 설정을 할 수 있습니다. 영어단어 난이도, 갯수, 시간제한 등을 설정하며 이어폰에 대한 소리 설정을 할 수 있습니다. 이는 만약 이어폰을 연결했을 경우 소리가 이어폰에서 나올지 아니면 핸드폰 스피커에서 나올지를 설정 할 수 있습니다.
-3) 알람 추가 및 변경 화면 : 메인 화면에 있던 알람 아이템을 클릭 할 경우 알람 시간과 요일 그리고 알람 이름을 설정할 수 있습니다. 또한 사용자가 알람이 울릴떄 진동으로 할지 소리로 할지를 설정 할 수 있습니다.
-4) 알람 해제 화면 : 알람이 울릴 경우 알람을 5분뒤로 설정하거나 혹은 알람해제를 통해 영어단어인식화면으로 전환합니다.
-5) 영단어 미션 화면 : 환경설정화면에서 정한 영단어 갯수, 난이도, 시간제한에 맞추어 미션이 시작되며 스피커버튼을 누른 발음대로 정확한 발음을 수행해야 미션해제 횟수를 증가시킬 수 있습니다.

개발 후기

이번 프로젝트에서 가장 신경써서 구현한 부분은 '알람 기능'이었습니다. 사용자가 알람을 등록할때 SQLite에 알람에 대한 정보(시간과 요일 라벨 등)와 AlarmReceiver에 해당 시간을 등록하게 하였습니다. 이때 Pendingintent를 사용하였고 intent에 알람에서 필요한정보(알람시간, 라벨, 요일, 소리설정 등)를 추가하여 send하였습니다. 각 pendingintent는 알람의 notificationId를 통해 구별하였습니다. 그리고 알람이 울릴 경우를 생각해보면 알람해제 버튼을 눌렀을때 다시 그 시간과 동일하게 AlarmReceiver에다가 등록을 하였고 5분 후 알람버튼을 누를 경우 AlarmReceiver 해당 시간 + 5min을 추가하여 등록을 하였습니다. 이때 flag변수를 함께 intent에 전송하여 flag변수가 true일 경우 알람해제시 AlarmReceiver에다가 등록을하지 않도록 구현하였습니다. (알람 중복 방지)
또한 알람기능을 구현하면서 정시에 알람이 울리게끔하는부분에서 많은 어려움이 있었습니다. 실제로 제 지인들 핸드폰을 빌려와 APK를 설치한 후 같은 시간대에 알람이 울리는 테스트를 시행한 결과 알람이 울리는 시간이 제각각이였고 저는 이러한 문제를 핸드폰 SDK마다 알람을 다르게 Set하여 해결하였습니다. 지금까지와는 다르게 알람부분은 Android 4대 컴포넌트 중 Broadcast와 Receiver를 주로 사용했기때문에 이해하기에 좀 더 시간이 걸렸지만 안드로이드에 기본에 좀 더 알아갈 수 있던 좋은 기회라고 생각합니다.

영어단어 인식의 경우 구글에서 제공하는 SpeechRecognize 클래스를 통해 좀더 쉽게 개발을 하였고 영어단어를 모두 DB에 저장하는것은 비효율적이라고 생각하여 공개 발행되있는 영어단어 엑셀을 불러와 I/O에 저장을 하는 방식으로 영어단어를 가져왔습니다. 또한 저는 항상 안드로이드 핸드폰을 사용해왔지만 아이폰 느낌의 UI를 원했 습니다. 그래서 저는 디자이너와 실제로 주 1-2회 컨택을 하면서 같이 개발을 진행하였습니다. 그 결과 저가 자부할 수 있을 정도로 퀄리티 있는 어플리케이션 디자인이 나올 수 있었습니다.
